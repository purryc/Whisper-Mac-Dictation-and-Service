# WhisperCPP Realtime ASR Gateway v0.2.0

## Release Summary

`v0.2.0` is the first formalized working release of the local WhisperCPP dictation and service stack.

This release includes:

- a reusable local gateway for Mac-hosted ASR
- Python and JavaScript SDKs for other projects
- a native macOS menu bar app with live captions
- dedicated start and stop command launchers
- machine-readable service discovery for other local Mac apps
- mixed-language improvements for Chinese and English

## Stable Local Workflow

1. Run `/Users/hmi/Documents/my skills/whispercpp-realtime-asr/WhisperCPP_install_runtime.command` once.
2. Run `/Users/hmi/Documents/my skills/whispercpp-realtime-asr/WhisperCPP_start_gateway.command`.
3. Open `/Users/hmi/Documents/my skills/whispercpp-realtime-asr/macos-app/dist/WhisperCppRealtimeMacApp.app`.
4. Use `Chinese + English` for Chinese-first mixed speech, `English` for English-only, and `Auto` only when you want unconstrained language detection.

## What Was Stabilized

### Gateway

- local `HTTP` and `WebSocket` reuse boundary
- local runtime installer
- dedicated gateway runner process
- service discovery file under `~/Library/Application Support/WhisperCPP/service.json`

### Native macOS app

- menu bar control center
- floating top-pinned captions
- global hotkey start and stop
- clean quit path
- auto-copy final transcript option
- crash fixes for audio callback isolation issues

### Mixed-language behavior

- bundled guidance moved from `ggml-base.bin` to `ggml-small.bin`
- app defaults to `Chinese + English`
- realtime session strategy now segments utterances on pauses

## Known Limits

- `Auto` mode can still turn Chinese into pinyin in some mixed-language sessions because this is a whisper.cpp behavior rather than a UI bug.
- The realtime strategy is intentionally lightweight and pause-based, not a full low-latency token streaming decoder.
- The repository does not commit local models, virtual environments, or exported `.app` bundles; those are rebuilt locally.
